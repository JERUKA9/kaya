Kaya is written in ruby using the Korundum bindings for KDE. Therefore, you need the ruby interpreter and said bindings to run it.

Kaya also includes some parts written in C++. To compile them, you need a KDE 4 development environment, and the ruby headers. On Ubuntu and similar systems, it should be enough to install kde-devel and ruby1.8-dev. Detailed installation instructions can be found in the INSTALL file.

Note that the C++ extension is optional, but some graphical effects are disabled when it is not installed.

Kaya may also be run without installing (using the kaya script in the root directory), with some caveats:

  • If you want the extension to be found, put the compiled library in the lib/ext directory.
  • KDE forces some files to be installed, namely the icons, the XMLGUI configuration file (kayaui.rc), and kaya.notifyrc. You might want to copy (or symlink) those files to the appropriate directories, or the corresponding features will be disabled or possibly misbehaving.
If you are hacking the code, or developing a plugin, running kaya in place is of course much more convenient than installing every time you make a change.
